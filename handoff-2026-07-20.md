## The website-agent upgrade (commit `39c36f3`) — what changed & why
Motivated by a tutorial Alex gave ("Build a $10K Website in Claude Code": Next.js + Framer Motion + a frontend-design skill + 21st.dev). That stack is React; our agent emits **vanilla static HTML/CSS/JS**, so I ported the *value*, not the literal stack. All additions are in `website_creator_agent.py`:

- **`DESIGN_SYSTEM`** constant (modular type scale, 8px grid, token-only color, anti-generic rules) injected into every stage's system prompt = the "frontend-design skill".
- **`SECTION_BLUEPRINTS`** constant (hero / feature grid / social proof / pricing / FAQ / CTA band / footer) shared by planner + page builder = the "21st.dev" equivalent.
- **Motion layer** (`MOTION_CSS` + rewritten `MAIN_JS`): IntersectionObserver scroll-reveals, staggered card entrances, hover polish, sticky-nav shadow, FAQ accordion = the "Framer Motion feel", vanilla. Respects `prefers-reduced-motion`; progressive-enhancement safe.
- **NEW cinematic scroll mode** (`--cinematic` CLI flag; auto-enabled in `create_website_for_chat` when a brief contains cinematic/flashy/immersive/etc.): a scroll-scrubbed "travel-in" hero — pinned full-screen scenes that zoom and hand off giant headlines, then release into the content payoff. Scroll engine is **deterministic** (`CINEMATIC_CSS` + `CINEMATIC_JS`); the model only writes the scene copy (`plan_cinema_scenes` → `SCENES_SCHEMA`). Assembled by `_render_cinema` / `_inject_cinema`.

**Three robustness fixes surfaced by verification (important — these are general, not cosmetic):**
- **`_fix_images`** — every missing local `<img>` is swapped for a styled `.media-ph` gradient placeholder carrying the alt text. No generated site ships a broken image. Real remote/`data:` images are kept.
- **`_balance_braces`** — repairs the two malformations model CSS actually produces: an unclosed brace, and a **truncated `var(--`** (unclosed paren). Either one silently swallowed every rule after it. (Observed live: a truncated `var(--` in `.card .card-icon` ate the entire effect layer.)
- **`effects.css` isolation** — the guaranteed motion + cinematic CSS is now written to its **own** `effects.css`, loaded *after* `styles.css`, and linked into every page via `_link_effects`. So no parse error in the model's `styles.css` can ever kill the effects again. **This was the key fix** that made the cinematic layer reliable.

### Honest limitations of the upgrade
- **The cinematic scenes are gradient PLACEHOLDERS**, not real footage. The agent generates no photos/video. To make a cinematic site actually look like the reference (Alex showed "Dapper Dave's" barber lounge), drop real images per scene: set `--scene-img: url('photoN.jpg')` on each `.cinema__bg`. Each scene already carries the ideal-shot description in an HTML comment + `aria-label`; the per-site README documents the swap.
- **I could not screen-capture the scroll motion** in the preview pane (it won't register programmatic scrolling on an ~11-viewport page). Verified instead via CSSOM (cinema rules apply, scenes `position: sticky`, scrub JS live) + rendered opening scene. Best confirmed by a human scrolling `sites/ironside-barber-lounge/` locally.
